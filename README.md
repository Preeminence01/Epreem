# EPREEM API (Laravel backend)

This is the Laravel 11 API that powers the EPREEM PWA frontend. It is API-only —
no Blade views — and authenticates the frontend via **Laravel Sanctum** (bearer tokens),
which works cleanly for both a browser PWA and a wrapped mobile app.

> **Important — this is an overlay, not a full Laravel skeleton.** The framework's
> generated scaffolding (the `artisan` executable, `public/index.php`, `config/app.php`,
> `config/auth.php`, `config/database.php`, `config/mail.php`, `config/logging.php`,
> `config/session.php`, `config/cache.php`, `.gitignore`, `phpunit.xml`, `storage/`
> and `bootstrap/cache/` directories, etc.) is generated by Composer and isn't
> reproduced here. **Start with a fresh skeleton first:**
> ```bash
> composer create-project laravel/laravel:^11.0 epreem-backend
> ```
> Then copy this project's `app/`, `database/`, `routes/`, `config/cors.php`,
> `config/sanctum.php`, `bootstrap/app.php`, and `.env.example` into that new project,
> overwriting the generated defaults. From there, the steps below apply normally.

## 1. Requirements

- PHP 8.2+
- Composer
- MySQL 8+ (or MariaDB)
- Node not required for the backend itself

## 2. Install

```bash
composer install
cp .env.example .env
php artisan key:generate
```

Edit `.env`:
- Set `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD` for your MySQL instance.
- Set `FRONTEND_URL` to wherever the EPREEM frontend is served from
  (e.g. `http://localhost:8080` while developing, your real domain in production).
- Set `SANCTUM_STATEFUL_DOMAINS` if you're using cookie-based auth in the browser;
  the frontend as shipped uses bearer tokens instead, so this matters less.

## 3. Database

```bash
php artisan migrate --seed
```

This creates all tables and seeds:
- 1 admin (`admin@epreem.com` / `password`)
- 1 buyer (`buyer@epreem.com` / `password`)
- 7 verified sellers/businesses, one per marketplace category
- 12 demo products (matching the frontend mockups), 3 of them live auctions
- Category commission rates
- A `EPREEM10` coupon ($1,000 off)

## 4. Run it

```bash
php artisan serve
# API now live at http://127.0.0.1:8000/api
```

Point the frontend's `js/config.js` `API_BASE_URL` at this address.

## 5. Storage (product images)

```bash
php artisan storage:link
```

Uploaded images are expected at `storage/app/public` and served from `/storage/...`.
The `ProductController@store` endpoint currently accepts product **data**; wire a
follow-up `POST /api/seller/products/{product}/images` endpoint (multipart upload,
`Intervention\Image` is already in composer.json for resizing) when you're ready to
support real image uploads from the seller dashboard's drag-and-drop box.

## 6. API surface

All routes are prefixed `/api`. See `routes/api.php` — it's grouped by capability
exactly as in the EPREEM URS:

| Group | Auth | Examples |
|---|---|---|
| Public / Guest | none | `GET /products`, `GET /products/{id}`, `GET /auctions`, `GET /categories` |
| Buyer | `auth:sanctum` | `/cart`, `/wishlist`, `/orders`, `/conversations`, bidding |
| Seller / Business | `auth:sanctum` + `role:seller` | `/seller/products`, `/seller/orders`, `/seller/stats` |
| Admin | `auth:sanctum` + `role:admin` | `/admin/users`, `/admin/listings`, `/admin/disputes`, `/admin/commissions` |

Auth flow: `POST /api/auth/register` or `/api/auth/login` returns `{ user, token }`.
Store the token (the frontend stores it in `localStorage` as `epreem_token`) and send
`Authorization: Bearer <token>` on every subsequent request.

Password reset: `POST /api/auth/forgot-password { email }` always returns a generic
success message (never reveals whether the address exists) and, if it does, writes a
reset link to `storage/logs/laravel.log` via the `log` mail driver. `POST
/api/auth/reset-password { email, token, password }` completes the reset. Swap in a
real mail driver and a `ResetPasswordMail` notification before going live — see the
`TODO` in `AuthController::forgotPassword`.

## 7. CORS

`config/cors.php` reads `FRONTEND_URL` from `.env` and allows credentials. If you serve
the PWA from multiple origins (e.g. staging + production), change `allowed_origins` to
an array literal instead of the single `env()` value.

## 8. Payments

`payment_status` is set to `paid` synchronously in `OrderController@store` for demo
purposes. Before going live, replace that block with a real gateway call (Paystack,
Flutterwave, or your mobile money aggregator of choice — key placeholders are already
in `.env.example`) and move status to `pending` until the webhook confirms payment.

## 9. Realtime (future)

Bid updates and new messages are written to the DB but not broadcast. For live
countdown/outbid updates without polling, add Laravel Reverb or Pusher and broadcast
`BidPlaced` / `MessageSent` events — the frontend's `js/app.js` already isolates all
network calls in `EpreemAPI`, so swapping polling for websockets later only touches
one file.
